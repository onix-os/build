# onix-rootasrole-policy

## Summary

Bootstrap factory policy for ONIX's RootAsRole privilege path.

The `rootasrole` package installs the binaries and safe defaults. This package
is the first package that owns the factory source for the live machine policy.

## System role

- Group: `services`
- Installed plane: machine/system policy
- Why ONIX needs it: RootAsRole is unusable as a real privilege tool until the
  machine has `/etc/security/rootasrole.json` and `/etc/pam.d/dosr`. Boulder
  packages `/usr`, so this stone owns the factory source under
  `/usr/share/factory/etc`.

## Implementation choice

- Implementation language: policy/data package
- Rust alternative considered: not applicable
- Serious Rust implementation exists: not applicable
- Selected implementation: package-owned RootAsRole JSON + PAM policy
- Why this implementation: privilege policy is machine state, so ONIX keeps it

## Runtime dependencies

Expected package dependencies:

```text
rootasrole
linux-pam
```

`rootasrole` pulls the `dosr` and `chsr` binaries. `linux-pam` provides the PAM
module surface used by `/etc/pam.d/dosr`.

## Installed paths

```text
/usr/share/factory/etc/security/rootasrole.json
/usr/share/factory/etc/pam.d/dosr
/usr/share/onix/packages/onix-rootasrole-policy.md
```

## Bootstrap policy

The first live policy is conservative:

- only `root` is an actor;
- `root` can run the root task;
- `root` can run `chsr`;
- normal users do not receive admin rights yet.

This lets ONIX prove policy materialization without silently granting the
default login user full machine privilege.

## Stone ownership

The factory policy files are owned by this stone directly. They are not copied
from Nix, and they are not generated by image assembly. A later image or system
materialization step copies the factory files into live `/etc`.
