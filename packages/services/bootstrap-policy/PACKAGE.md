# bootstrap-policy

## Summary

Bootstrap helper scripts, proof notes, and source copies of early ONIX systemd
service units.

## System role

- Group: `services`
- Installed plane: machine/system service policy
- Why ONIX needs it: keeps bootstrap service policy package-owned instead of
  hidden inside image assembly shell heredocs.

## Implementation choice

- Implementation language: POSIX shell scripts plus systemd unit files
- Rust alternative considered: not for this phase
- Serious Rust implementation exists: not applicable for static service policy
- Selected implementation: shell scripts and unit files
- Why this implementation: the package records early boot proof helpers and
  unit source policy. These are small glue files, not long-lived ONIX daemons.

If a helper becomes a real ONIX tool later, it should be rewritten in Rust.

## Source and provenance

- Upstream: ONIX project
- Source archive or repository: generated by current Phase 4 bootstrap-policy
  packaging flow
- Pinned version: recipe placeholder `@BOOTSTRAP_POLICY_VERSION@`
- Source hash: recipe placeholder `@BOOTSTRAP_POLICY_PAYLOAD_SHA256@`
- Patch set: Phase 4 bootstrap policy source state

## Build model

- Build environment: Boulder recipe install phase
- Build tools: POSIX shell utilities from the builder
- Target triple: not applicable
- C runtime: not applicable
- Link model: scripts and unit files
- Shared runtime libraries: none

## Runtime-clean contract

- No runtime `/nix/store` dependency: yes for active helpers and units
- No `/nix/store` shebangs: yes
- No `/nix/store` RPATH/RUNPATH: yes
- No systemd units calling `/nix/store`: yes
- No glibc loader path: yes
- No unexpected shared runtime libraries: yes

## Runtime dependencies

```text
- /usr/bin/busybox
  reason: shell/helper execution during bootstrap proofs
  owner package: busybox
- /usr/sbin/dropbear
  reason: SSH bootstrap service
  owner package: dropbear
- /usr/lib/systemd/systemd
  reason: service manager
  owner package: systemd
```

## Installed paths

```text
/usr/lib/onix/bootstrap-*
/usr/lib/onix/systemd/system/onix-bootstrap-*.service
/usr/share/onix/bootstrap/*.txt
/usr/share/onix/packages/bootstrap-policy.md
```

## Stone ownership

The finished `.stone` owns bootstrap policy files directly. Phase 4 may still
materialize active units into systemd search paths.

## Exceptions

This package is bootstrap policy. Long-lived ONIX tools should not remain shell
glue if they grow beyond small proof helpers.
